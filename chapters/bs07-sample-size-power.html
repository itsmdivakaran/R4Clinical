16  Sample Size and Power Calculations

16.1 Introduction

Sample size determination is one of the most consequential decisions in clinical trial design. An underpowered study risks missing a real treatment effect; an overpowered study wastes resources and exposes more patients than necessary to experimental treatments. Regulatory agencies require prospective justification of sample size in every confirmatory trial protocol.

This chapter covers:

  • Fundamental concepts: Effect size, power, Type I and II errors
  • Two-sample t-test: Continuous endpoints
  • Two proportions: Binary endpoints
  • Survival endpoints: Log-rank test and event-driven designs
  • Equivalence and non-inferiority: TOST and one-sided designs
  • Multiple comparisons: Adjusting for multiplicity
  • Sensitivity analyses: Handling uncertainty in design assumptions
  • R packages: pwr, pwrss, gsDesign, rpact

16.2 1. Core Concepts

16.2.1 1.1 The Four Design Parameters

Any sample size calculation involves four interdependent quantities. Fix any three and the fourth is determined:

Parameter Symbol Typical value
Significance level (Type I error rate) \(\alpha\) 0.05 (two-sided)
Power (1 − Type II error rate) \(1-\beta\) 0.80 or 0.90
Effect size (standardised or absolute) \(\delta\) Clinically meaningful difference
Sample size \(n\) To be determined 
library(tidyverse)
library(pwr)
library(gt)
set.seed(42)

# Illustrate power as a function of sample size
alpha  <- 0.05
delta  <- 0.5    # medium standardised effect (Cohen's d)
n_vals <- seq(10, 300, by = 5)

power_vals <- sapply(n_vals, function(n) {
  pwr.t.test(n = n, d = delta, sig.level = alpha,
             type = "two.sample", alternative = "two.sided")$power
})

tibble(n = n_vals, power = power_vals) |>
  ggplot(aes(x = n, y = power)) +
  geom_line(colour = "#2C6E9E", linewidth = 1.1) +
  geom_hline(yintercept = 0.80, linetype = "dashed", colour = "#D32F2F") +
  geom_hline(yintercept = 0.90, linetype = "dotted", colour = "#E07B39") +
  geom_vline(
    xintercept = pwr.t.test(d = delta, sig.level = alpha,
                             power = 0.80, type = "two.sample")$n,
    linetype = "dashed", colour = "#D32F2F", alpha = 0.7
  ) +
  annotate("text", x = 5, y = 0.82, label = "80% power",
           colour = "#D32F2F", hjust = 0, size = 3.5) +
  annotate("text", x = 5, y = 0.92, label = "90% power",
           colour = "#E07B39", hjust = 0, size = 3.5) +

    y     = "Power",

  ) +
  theme_bw(base_size = 12)

16.3 2. Sample Size for Continuous Endpoints

16.3.1 2.1 Two-Sample t-Test

The required sample size per arm:

\[n = \frac{2\sigma^2(z_{\alpha/2} + z_\beta)^2}{\delta^2}\]

where \(\delta\) is the mean difference, \(\sigma\) is the common standard deviation.

# Example: Systolic blood pressure trial
# MCID = 5 mmHg reduction; SD = 15 mmHg; α = 0.05 (two-sided); 90% power

delta_sbp <- 5       # MCID: 5 mmHg
sigma_sbp <- 15      # assumed SD
alpha_sbp <- 0.05
power_sbp <- 0.90

ss_sbp <- pwr.t.test(
  d          = delta_sbp / sigma_sbp,   # Cohen's d
  sig.level  = alpha_sbp,
  power      = power_sbp,
  type       = "two.sample",
  alternative = "two.sided"
)

n_sbp <- ceiling(ss_sbp$n)

cat("=== SBP Trial Sample Size ===\n")
#> === SBP Trial Sample Size ===
cat("MCID (δ):      ", delta_sbp, "mmHg\n")
#> MCID (δ):       5 mmHg
cat("Assumed SD:    ", sigma_sbp, "mmHg\n")
#> Assumed SD:     15 mmHg
cat("Cohen's d:     ", round(delta_sbp / sigma_sbp, 3), "\n")
#> Cohen's d:      0.333
cat("α (two-sided): ", alpha_sbp, "\n")
#> α (two-sided):  0.05
cat("Power (1-β):   ", power_sbp, "\n")
#> Power (1-β):    0.9
cat("n per arm:     ", n_sbp, "\n")
#> n per arm:      191
cat("Total N:       ", n_sbp * 2, "\n\n")
#> Total N:        382
# Sensitivity: vary SD assumption
cat("Sensitivity to SD assumption:\n")
#> Sensitivity to SD assumption:
tibble(
  SD       = c(10, 12, 15, 18, 20),
  n_per_arm = sapply(c(10, 12, 15, 18, 20), function(sd) {

  }),
  Total_N  = sapply(c(10, 12, 15, 18, 20), function(sd) {

16.4 3. Sample Size for Binary Endpoints

16.4.1 3.1 Two-Proportion z-Test

For comparing two proportions \(p_1\) and \(p_2\):

\[n = \frac{(z_{\alpha/2}\sqrt{2\bar{p}(1-\bar{p})} + z_\beta\sqrt{p_1(1-p_1) + p_2(1-p_2)})^2}{(p_1 - p_2)^2}\]

where \(\bar{p} = (p_1 + p_2)/2\).

# Example: response rate trial
p1 <- 0.55   # expected response rate (Active)
p2 <- 0.40   # expected response rate (Placebo)
alpha_bin <- 0.05
power_bin <- 0.80

ss_bin <- pwr.2p.test(
  h          = ES.h(p1, p2),
  sig.level  = alpha_bin,
  power      = power_bin,
  alternative = "two.sided"
)
n_bin <- ceiling(ss_bin$n)

cat("=== Binary Endpoint Trial (Response Rate) ===\n")
#> === Binary Endpoint Trial (Response Rate) ===
cat("Active response rate (p1):  ", p1, "\n")
#> Active response rate (p1):   0.55
cat("Placebo response rate (p2): ", p2, "\n")
#> Placebo response rate (p2):  0.4
cat("Difference:                 ", p1 - p2, "(", (p1 - p2) * 100, "pp)\n")
#> Difference:                  0.15 ( 15 pp)
cat("Cohen's h:                  ", round(ES.h(p1, p2), 4), "\n")
#> Cohen's h:                   0.3015
cat("n per arm:                  ", n_bin, "\n")
#> n per arm:                   173
cat("Total N:                    ", n_bin * 2, "\n\n")
#> Total N:                     346

16.5 4. Sample Size for Survival Endpoints

16.5.1 4.1 Event-Driven Designs

Survival trials are powered by the number of events (deaths, progressions), not the number of subjects. The required number of events for the log-rank test:

\[E = \frac{4(z_{\alpha/2} + z_\beta)^2}{[\log(HR)]^2}\]

# Survival trial: OS in oncology
# HR = 0.70 (30% reduction in hazard); α = 0.05; power = 80%
hr      <- 0.70
alpha_s <- 0.05
power_s <- 0.80

# Required events (Schoenfeld formula)
z_alpha <- qnorm(1 - alpha_s / 2)
z_beta  <- qnorm(power_s)
e_req   <- ceiling(4 * (z_alpha + z_beta)^2 / (log(hr))^2)

cat("=== Survival Trial (Overall Survival) ===\n")
#> === Survival Trial (Overall Survival) ===
cat("Hazard Ratio (HR):     ", hr, "\n")
#> Hazard Ratio (HR):      0.7
cat("α (two-sided):         ", alpha_s, "\n")
#> α (two-sided):          0.05
cat("Power (1-β):           ", power_s, "\n")
#> Power (1-β):            0.8
cat("Required events:       ", e_req, "\n\n")
#> Required events:        247
# Translate events to subjects (assuming event probability)
# Accrual = 24 months, follow-up = 12 months, control median OS = 18 months
lambda_ctrl   <- log(2) / 18        # control arm hazard (median 18 months)
lambda_active <- lambda_ctrl * hr   # active arm hazard

# Probability of event by end of trial (36 months max)
t_max <- 36
# Simplified: event probability ~ 1 - exp(-lambda * t_max) averaged over uniform accrual
p_event_ctrl   <- 1 - exp(-lambda_ctrl   * t_max * 0.6)  # approximate
p_event_active <- 1 - exp(-lambda_active * t_max * 0.6)
p_event_avg    <- (p_event_ctrl + p_event_active) / 2

n_subjects <- ceiling(e_req / p_event_avg)

cat("Event probability (approx, per arm): ", round(p_event_avg, 3), "\n")
#> Event probability (approx, per arm):  0.503
cat("Estimated subjects per arm:          ", ceiling(n_subjects / 2), "\n")
#> Estimated subjects per arm:           246
cat("Total subjects:                      ", n_subjects, "\n\n")
#> Total subjects:                       492
# HR sensitivity table
hr_vals <- c(0.60, 0.65, 0.70, 0.75, 0.80)
tibble(
  HR           = hr_vals,
  Events_req   = sapply(hr_vals, function(h) {
    ceiling(4 * (qnorm(0.975) + qnorm(0.80))^2 / (log(h))^2)
  })
) |>
  gt() |>
  tab_header(title = "Events Required vs. Hazard Ratio",
             subtitle = "α = 0.05 (two-sided), power = 80%")

16.6 5. Non-Inferiority and Equivalence Trials

16.6.1 5.1 Non-Inferiority Margin

Non-inferiority (NI) trials test that a new treatment is not worse than an active comparator by more than a pre-specified margin \(\Delta_{NI}\).

\[H_0: \mu_{\text{test}} - \mu_{\text{ref}} \leq -\Delta_{NI}\] \[H_1: \mu_{\text{test}} - \mu_{\text{ref}} > -\Delta_{NI}\]

# Non-inferiority: new formulation vs. reference
# NI margin = -5 mmHg; expected true difference = 0; SD = 12
delta_ni  <- 0     # true difference (expected: no difference)
margin_ni <- 5     # NI margin (mmHg)
sigma_ni  <- 12
alpha_ni  <- 0.025  # one-sided alpha
power_ni  <- 0.90

# For NI test: n = 2σ²(z_α + z_β)² / (margin - delta)²
z_a <- qnorm(1 - alpha_ni)
z_b <- qnorm(power_ni)
n_ni <- ceiling(2 * sigma_ni^2 * (z_a + z_b)^2 / (margin_ni - delta_ni)^2)

cat("=== Non-Inferiority Trial ===\n")
#> === Non-Inferiority Trial ===
cat("NI margin (Δ):         ", margin_ni, "mmHg\n")
#> NI margin (Δ):          5 mmHg
cat("Expected difference:   ", delta_ni, "mmHg (no difference)\n")
#> Expected difference:    0 mmHg (no difference)
cat("σ:                     ", sigma_ni, "\n")
#> σ:                      12
cat("α (one-sided):         ", alpha_ni, "\n")
#> α (one-sided):          0.025
cat("Power (1-β):           ", power_ni, "\n")
#> Power (1-β):            0.9
cat("n per arm:             ", n_ni, "\n")
#> n per arm:              122
cat("Total N:               ", n_ni * 2, "\n\n")
#> Total N:                244
# Compare with superiority design
n_sup <- ceiling(pwr.t.test(
  d         = 5 / 12,
  sig.level = 0.05,
  power     = 0.90,
  type      = "two.sample"
)$n)

cat("Equivalent superiority design (δ = 5 mmHg):\n")
#> Equivalent superiority design (δ = 5 mmHg):
cat("  n per arm:", n_sup, "| Total:", n_sup * 2, "\n")
#>   n per arm: 123 | Total: 246
cat("  (NI trials often need fewer patients since margin > 0)\n")
#>   (NI trials often need fewer patients since margin > 0)

16.6.2 5.2 Two One-Sided Tests (TOST) for Equivalence

# Bioequivalence: 80-125% rule (on log scale)
# For log-transformed PK parameters: log(1.25) ≈ 0.223
margin_be <- log(1.25)
sigma_be  <- 0.25      # CV ≈ 25% for PK data
power_be  <- 0.90
alpha_be  <- 0.05

n_be <- ceiling(
  2 * sigma_be^2 * (qnorm(1 - alpha_be) + qnorm(power_be))^2 / margin_be^2
)

cat("=== Bioequivalence (TOST) ===\n")
#> === Bioequivalence (TOST) ===
cat("BE margin:    ±", round(margin_be, 4), "(log scale) = 80–125% ratio\n")
#> BE margin:    ± 0.2231 (log scale) = 80–125% ratio
cat("CV (σ):       ", sigma_be, "\n")
#> CV (σ):        0.25
cat("Power:        ", power_be, "\n")
#> Power:         0.9
cat("Sequences×periods (crossover): 2 × 2\n")
#> Sequences×periods (crossover): 2 × 2
cat("n subjects:   ", n_be, "\n")
#> n subjects:    22

16.8 7. Adaptive and Group Sequential Designs

16.8.1 7.1 Group Sequential Designs

Group sequential designs allow pre-planned interim analyses with the option to stop early for efficacy or futility, while controlling the overall Type I error rate.

# O'Brien-Fleming boundaries for 2 interim + final analysis
# Using simple z-score boundaries (illustrative)
n_analyses   <- 3     # 2 interim + 1 final
alpha_total  <- 0.05

# Pocock spending function (constant boundaries)
z_pocock <- qnorm(1 - alpha_total / (2 * n_analyses))

# O'Brien-Fleming spending (conservative at interim, liberal at final)
info_times <- c(1/3, 2/3, 1)   # information fraction at each analysis
of_bounds <- qnorm(1 - alpha_total / 2) / sqrt(info_times)

cat("=== Group Sequential Boundaries (Illustrative) ===\n\n")
#> === Group Sequential Boundaries (Illustrative) ===
tibble(
  Analysis       = c("Interim 1", "Interim 2", "Final"),
  Info_fraction  = info_times,
  n_analysed     = ceiling(info_times * n_sbp * 2),
  Pocock_z       = round(rep(z_pocock, 3), 3),
  OBF_z          = round(of_bounds, 3),
  Pocock_p       = round(2 * pnorm(-z_pocock), 4),
  OBF_p          = round(2 * pnorm(-of_bounds), 4)
) |>
  gt() |>
  tab_header(
    title    = "Group Sequential Efficacy Boundaries",
    subtitle = "3-analysis design, α = 0.05 (two-sided)"
  ) |>
  cols_label(
    Info_fraction = "Information Fraction",
    n_analysed    = "N at Analysis",
    Pocock_z      = "Pocock z",

16.10 Summary

Rigorous sample size calculation is a regulatory requirement and a scientific obligation. Key principles:

  • Four parameters — \(\alpha\), power, effect size, and \(n\) — are interdependent: specify three, solve for the fourth
  • The effect size is the most consequential input: use the MCID, not an optimistic estimate, and conduct sensitivity analyses
  • Standard designs: two-sample t-test for continuous endpoints; two-proportion z-test for binary; log-rank for survival
  • Event-driven survival trials: power depends on the number of events, not just the sample size; recruitment and follow-up periods determine when events accumulate
  • Non-inferiority and equivalence trials use one-sided tests and require a pre-specified, scientifically justified NI margin
  • Inflate for dropout: always account for expected withdrawal when calculating the number to enrol
  • Group sequential designs allow for early stopping while maintaining overall error control
  • Pre-registration: sample size justification must be documented in the protocol and SAP before data collection begins

Regulatory note: ICH E9 requires that the sample size justification be included in the protocol, with explicit statements of the primary endpoint, assumed treatment difference, SD (or event rate), \(\alpha\), and power. Sensitivity analyses exploring the impact of varying assumptions are strongly recommended.

# Sample Size and Power Calculations

## Introduction

Sample size determination is one of the most consequential decisions in clinical trial design. An underpowered study risks missing a real treatment effect; an overpowered study wastes resources and exposes more patients than necessary to experimental treatments. Regulatory agencies require prospective justification of sample size in every confirmatory trial protocol.

This chapter covers:

- **Fundamental concepts**: Effect size, power, Type I and II errors
- **Two-sample t-test**: Continuous endpoints
- **Two proportions**: Binary endpoints
- **Survival endpoints**: Log-rank test and event-driven designs
- **Equivalence and non-inferiority**: TOST and one-sided designs
- **Multiple comparisons**: Adjusting for multiplicity
- **Sensitivity analyses**: Handling uncertainty in design assumptions
- **R packages**: `pwr`, `pwrss`, `gsDesign`, `rpact`

---

## 1. Core Concepts

### 1.1 The Four Design Parameters

Any sample size calculation involves four interdependent quantities. Fix any three and the fourth is determined:

| Parameter | Symbol | Typical value |
|---|---|---|
| Significance level (Type I error rate) | $\alpha$ | 0.05 (two-sided) |
| Power (1 − Type II error rate) | $1-\beta$ | 0.80 or 0.90 |
| Effect size (standardised or absolute) | $\delta$ | Clinically meaningful difference |
| Sample size | $n$ | To be determined |


## 2. Sample Size for Continuous Endpoints

### 2.1 Two-Sample t-Test

The required sample size per arm:

$$n = \frac{2\sigma^2(z_{\alpha/2} + z_\beta)^2}{\delta^2}$$

where $\delta$ is the mean difference, $\sigma$ is the common standard deviation.

```{r}
#| label: ss-continuous

# Example: Systolic blood pressure trial
# MCID = 5 mmHg reduction; SD = 15 mmHg; α = 0.05 (two-sided); 90% power

delta_sbp <- 5       # MCID: 5 mmHg
sigma_sbp <- 15      # assumed SD
alpha_sbp <- 0.05
power_sbp <- 0.90

ss_sbp <- pwr.t.test(
  d          = delta_sbp / sigma_sbp,   # Cohen's d
  sig.level  = alpha_sbp,
  power      = power_sbp,
  type       = "two.sample",
  alternative = "two.sided"
)

n_sbp <- ceiling(ss_sbp$n)

cat("=== SBP Trial Sample Size ===\n")
cat("MCID (δ):      ", delta_sbp, "mmHg\n")
cat("Assumed SD:    ", sigma_sbp, "mmHg\n")
cat("Cohen's d:     ", round(delta_sbp / sigma_sbp, 3), "\n")
cat("α (two-sided): ", alpha_sbp, "\n")
cat("Power (1-β):   ", power_sbp, "\n")
cat("n per arm:     ", n_sbp, "\n")
cat("Total N:       ", n_sbp * 2, "\n\n")

# Sensitivity: vary SD assumption
cat("Sensitivity to SD assumption:\n")
tibble(
  SD       = c(10, 12, 15, 18, 20),
  n_per_arm = sapply(c(10, 12, 15, 18, 20), function(sd) {
    ceiling(pwr.t.test(d = delta_sbp / sd, sig.level = alpha_sbp,
                       power = power_sbp, type = "two.sample")$n)
  }),
  Total_N  = sapply(c(10, 12, 15, 18, 20), function(sd) {

                            power = power_sbp, type = "two.sample")$n)
  })
) |>
```

## 3. Sample Size for Binary Endpoints

### 3.1 Two-Proportion z-Test

For comparing two proportions $p_1$ and $p_2$:

$$n = \frac{(z_{\alpha/2}\sqrt{2\bar{p}(1-\bar{p})} + z_\beta\sqrt{p_1(1-p_1) + p_2(1-p_2)})^2}{(p_1 - p_2)^2}$$

where $\bar{p} = (p_1 + p_2)/2$.

```{r}
#| label: ss-binary

# Example: response rate trial
p1 <- 0.55   # expected response rate (Active)
p2 <- 0.40   # expected response rate (Placebo)
alpha_bin <- 0.05
power_bin <- 0.80

ss_bin <- pwr.2p.test(
  h          = ES.h(p1, p2),
  sig.level  = alpha_bin,
  power      = power_bin,
  alternative = "two.sided"
)
n_bin <- ceiling(ss_bin$n)

cat("=== Binary Endpoint Trial (Response Rate) ===\n")
cat("Active response rate (p1):  ", p1, "\n")
cat("Placebo response rate (p2): ", p2, "\n")
cat("Difference:                 ", p1 - p2, "(", (p1 - p2) * 100, "pp)\n")
cat("Cohen's h:                  ", round(ES.h(p1, p2), 4), "\n")
cat("n per arm:                  ", n_bin, "\n")
cat("Total N:                    ", n_bin * 2, "\n\n")

# Sensitivity: vary response rates
```

## 4. Sample Size for Survival Endpoints

### 4.1 Event-Driven Designs

Survival trials are powered by the **number of events** (deaths, progressions), not the number of subjects. The required number of events for the log-rank test:

$$E = \frac{4(z_{\alpha/2} + z_\beta)^2}{[\log(HR)]^2}$$

```{r}
#| label: ss-survival

# Survival trial: OS in oncology
# HR = 0.70 (30% reduction in hazard); α = 0.05; power = 80%
hr      <- 0.70
alpha_s <- 0.05
power_s <- 0.80

# Required events (Schoenfeld formula)
z_alpha <- qnorm(1 - alpha_s / 2)
z_beta  <- qnorm(power_s)
e_req   <- ceiling(4 * (z_alpha + z_beta)^2 / (log(hr))^2)

cat("=== Survival Trial (Overall Survival) ===\n")
cat("Hazard Ratio (HR):     ", hr, "\n")
cat("α (two-sided):         ", alpha_s, "\n")
cat("Power (1-β):           ", power_s, "\n")
cat("Required events:       ", e_req, "\n\n")

# Translate events to subjects (assuming event probability)
# Accrual = 24 months, follow-up = 12 months, control median OS = 18 months
lambda_ctrl   <- log(2) / 18        # control arm hazard (median 18 months)
lambda_active <- lambda_ctrl * hr   # active arm hazard

# Probability of event by end of trial (36 months max)
t_max <- 36
# Simplified: event probability ~ 1 - exp(-lambda * t_max) averaged over uniform accrual
p_event_ctrl   <- 1 - exp(-lambda_ctrl   * t_max * 0.6)  # approximate
p_event_active <- 1 - exp(-lambda_active * t_max * 0.6)
p_event_avg    <- (p_event_ctrl + p_event_active) / 2

n_subjects <- ceiling(e_req / p_event_avg)

cat("Event probability (approx, per arm): ", round(p_event_avg, 3), "\n")
cat("Estimated subjects per arm:          ", ceiling(n_subjects / 2), "\n")
cat("Total subjects:                      ", n_subjects, "\n\n")

# HR sensitivity table
hr_vals <- c(0.60, 0.65, 0.70, 0.75, 0.80)
tibble(
  HR           = hr_vals,
  Events_req   = sapply(hr_vals, function(h) {
    ceiling(4 * (qnorm(0.975) + qnorm(0.80))^2 / (log(h))^2)
  })
) |>
  gt() |>
  tab_header(title = "Events Required vs. Hazard Ratio",
             subtitle = "α = 0.05 (two-sided), power = 80%")
```

## 5. Non-Inferiority and Equivalence Trials

### 5.1 Non-Inferiority Margin

Non-inferiority (NI) trials test that a new treatment is not worse than an active comparator by more than a pre-specified margin $\Delta_{NI}$.

$$H_0: \mu_{\text{test}} - \mu_{\text{ref}} \leq -\Delta_{NI}$$
$$H_1: \mu_{\text{test}} - \mu_{\text{ref}} > -\Delta_{NI}$$

```{r}
#| label: ss-ni

# Non-inferiority: new formulation vs. reference
# NI margin = -5 mmHg; expected true difference = 0; SD = 12
delta_ni  <- 0     # true difference (expected: no difference)
margin_ni <- 5     # NI margin (mmHg)
sigma_ni  <- 12
alpha_ni  <- 0.025  # one-sided alpha
power_ni  <- 0.90

# For NI test: n = 2σ²(z_α + z_β)² / (margin - delta)²
z_a <- qnorm(1 - alpha_ni)
z_b <- qnorm(power_ni)
n_ni <- ceiling(2 * sigma_ni^2 * (z_a + z_b)^2 / (margin_ni - delta_ni)^2)

cat("=== Non-Inferiority Trial ===\n")
cat("NI margin (Δ):         ", margin_ni, "mmHg\n")
cat("Expected difference:   ", delta_ni, "mmHg (no difference)\n")
cat("σ:                     ", sigma_ni, "\n")
cat("α (one-sided):         ", alpha_ni, "\n")
cat("Power (1-β):           ", power_ni, "\n")
cat("n per arm:             ", n_ni, "\n")
cat("Total N:               ", n_ni * 2, "\n\n")

# Compare with superiority design
n_sup <- ceiling(pwr.t.test(
  d         = 5 / 12,
  sig.level = 0.05,
  power     = 0.90,
  type      = "two.sample"
)$n)

cat("Equivalent superiority design (δ = 5 mmHg):\n")
cat("  n per arm:", n_sup, "| Total:", n_sup * 2, "\n")
cat("  (NI trials often need fewer patients since margin > 0)\n")
```

## 7. Adaptive and Group Sequential Designs

### 7.1 Group Sequential Designs

Group sequential designs allow pre-planned interim analyses with the option to stop early for efficacy or futility, while controlling the overall Type I error rate.

## Summary

Rigorous sample size calculation is a regulatory requirement and a scientific obligation. Key principles:

- **Four parameters** — $\alpha$, power, effect size, and $n$ — are interdependent: specify three, solve for the fourth
- **The effect size is the most consequential input**: use the MCID, not an optimistic estimate, and conduct sensitivity analyses
- **Standard designs**: two-sample t-test for continuous endpoints; two-proportion z-test for binary; log-rank for survival
- **Event-driven survival trials**: power depends on the number of events, not just the sample size; recruitment and follow-up periods determine when events accumulate
- **Non-inferiority and equivalence trials** use one-sided tests and require a pre-specified, scientifically justified NI margin
- **Inflate for dropout**: always account for expected withdrawal when calculating the number to enrol
- **Group sequential designs** allow for early stopping while maintaining overall error control
- **Pre-registration**: sample size justification must be documented in the protocol and SAP before data collection begins

> **Regulatory note**: ICH E9 requires that the sample size justification be included in the protocol, with explicit statements of the primary endpoint, assumed treatment difference, SD (or event rate), $\alpha$, and power. Sensitivity analyses exploring the impact of varying assumptions are strongly recommended.
