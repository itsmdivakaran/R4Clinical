4  Data Cleaning and Transformation in Clinical Research

4.1 4.1 The Reality of Clinical Data Quality

“Garbage in, garbage out” is nowhere more true than in clinical research. Raw clinical data from electronic health records, clinical trial databases, and lab systems is messy. It contains:

  • Missing values: Incomplete lab results, unreported adverse events, patients lost to follow-up
  • Outliers and implausible values: A glucose of 1,500 mg/dL (certainly an error); a weight of 5 kg (impossible for an adult)
  • Inconsistencies: Multiple names for the same adverse event (“Headache”, “Headaches”, “HEAD PAIN”)
  • Format inconsistencies: Dates as “3/15/2024”, “03-15-2024”, and “2024-03-15” in the same column
  • Duplicates: Same lab result recorded twice, multiple records for the same patient visit
  • Reference range issues: Lab units that changed mid-study, different reference ranges for different labs

In clinical research, data quality is not just an analytical concern—it’s a regulatory requirement. The FDA expects data to be “clean” before statistical analysis. Your role is to identify problems and either correct them (if the error is obvious) or flag them for the clinical team to resolve.

Data Quality is a Team Sport: You (as the analyst) are responsible for identifying and documenting data quality issues. The clinical team, data manager, and medical monitor are responsible for investigating root causes and deciding on corrections. Never silently fix data without documenting the change.

4.2.2 Missing Data Mechanism

Understanding why data is missing matters statistically:

MCAR (Missing Completely At Random): Missing values are unrelated to the data itself. Example: A technician accidentally didn’t record one lab value. Missingness is random.

MAR (Missing At Random): Missingness depends on other observed variables but not on the missing value itself. Example: Labs weren’t done in subjects who discontinued early, but we know when they discontinued (observed).

MNAR (Missing Not At Random): Missingness depends on the missing value itself. Example: Subjects with very high glucose may be less likely to have follow-up labs recorded (sicker patients). The missingness itself is informative.

library(tidyverse)

data("lb")

Pairwise Deletion

Use all available data for each analysis. Default for many statistical functions (e.g., cor() with use = "complete.obs").

Mean/Median Imputation

Replace missing values with the mean or median of non-missing values. Simple but reduces variance.

library(tidyverse)

Imputation Concerns: Imputing missing values reduces variance, biases standard errors, and can introduce bias if missingness is MNAR. In clinical trials, imputation methods must be pre-specified in the Statistical Analysis Plan. Simple mean imputation is rarely appropriate; consider multiple imputation ({mice} package) for more sophisticated approaches.

Last Observation Carried Forward (LOCF)

For longitudinal data, use the last non-missing value as a proxy for later visits. Used in clinical trials but increasingly scrutinized.

library(tidyverse)

lab_visits <- tibble(
  USUBJID = c("001-101", "001-101", "001-101", "001-101",
              "001-102", "001-102", "001-102", "001-102"),

  group_by(USUBJID) %>%

      lag(HGB, n = 1, order_by = VISIT),

Multiple Imputation

Create multiple “plausible” completed datasets, analyze each separately, and pool results. More sophisticated and accounts for uncertainty in imputation.

# mice package: multiple imputation by chained equations
if (requireNamespace("mice", quietly = TRUE)) {
  # Create a dataset with missing values
  incomplete_data <- data.frame(
    USUBJID   = paste0("001-", 101:110),
    AGE       = c(62, 45, NA, 56, 31, 54, 46, 34, 29, 35),
    WEIGHT_KG = c(70, 85, 92, NA, 60, 75, 68, NA, 77, 82),
    GLUCOSE   = c(95, 102, 110, 88, NA, 98, 105, 92, 87, 100)
  )
  # Multiple imputation
  imputed_data      <- mice::mice(incomplete_data, m = 5,
                                   method = "pmm", seed = 123, printFlag = FALSE)
  completed_datasets <- mice::complete(imputed_data, action = "all")
  results            <- lapply(completed_datasets,
                               function(d) lm(GLUCOSE ~ AGE + WEIGHT_KG, data = d))
  cat("Age coefficients across 5 imputed datasets:
")
  print(round(sapply(results, function(m) coef(m)["AGE"]), 3))
} else {
  message("mice not installed. Install with: install.packages('mice')")
  message("Demonstrating with complete cases only:")
  incomplete_data <- data.frame(
    AGE       = c(62, 45, 56, 31, 54, 46, 34, 29, 35),
    WEIGHT_KG = c(70, 85, 75, 60, 75, 68, 82, 77, 82),
    GLUCOSE   = c(95, 102, 88, NA, 98, 105, 92, 87, 100)
  )
  m <- lm(GLUCOSE ~ AGE + WEIGHT_KG, data = incomplete_data)
  print(coef(m))
}

Long format (also called “tidy” data):

Wide format:

USUBJID  VISIT  Hemoglobin  Glucose

  mutate(

  fill(BASELINE_VAL, .direction = "down") %>%
  ungroup() %>%

                  LBSTRESN - BASELINE_VAL,
                  NA_real_)
  ) %>%

  pivot_wider(

    names_from = LBTEST,

    names_sep = "_"

  # Step 5: Join with treatment information
  left_join(
    dm %>% select(USUBJID, ARM, AGE, SEX),
    by = "USUBJID"
  ) %>%

  select(USUBJID, ARM, AGE, SEX, LBDTC, VISIT, everything())

## 4.1 The Reality of Clinical Data Quality

"Garbage in, garbage out" is nowhere more true than in clinical research. Raw clinical data from electronic health records, clinical trial databases, and lab systems is messy. It contains:

- **Missing values**: Incomplete lab results, unreported adverse events, patients lost to follow-up
- **Outliers and implausible values**: A glucose of 1,500 mg/dL (certainly an error); a weight of 5 kg (impossible for an adult)
- **Inconsistencies**: Multiple names for the same adverse event ("Headache", "Headaches", "HEAD PAIN")
- **Format inconsistencies**: Dates as "3/15/2024", "03-15-2024", and "2024-03-15" in the same column
- **Duplicates**: Same lab result recorded twice, multiple records for the same patient visit
- **Reference range issues**: Lab units that changed mid-study, different reference ranges for different labs

In clinical research, data quality is not just an analytical concern—it's a regulatory requirement. The FDA expects data to be "clean" before statistical analysis. Your role is to identify problems and either correct them (if the error is obvious) or flag them for the clinical team to resolve.

::: {.callout-important}
**Data Quality is a Team Sport**: You (as the analyst) are responsible for identifying and documenting data quality issues. The clinical team, data manager, and medical monitor are responsible for investigating root causes and deciding on corrections. Never silently fix data without documenting the change.
:::

### Missing Data Mechanism

Understanding *why* data is missing matters statistically:

**MCAR (Missing Completely At Random)**: Missing values are unrelated to the data itself. Example: A technician accidentally didn't record one lab value. Missingness is random.

**MAR (Missing At Random)**: Missingness depends on other observed variables but not on the missing value itself. Example: Labs weren't done in subjects who discontinued early, but we know when they discontinued (observed).

**MNAR (Missing Not At Random)**: Missingness depends on the missing value itself. Example: Subjects with very high glucose may be less likely to have follow-up labs recorded (sicker patients). The missingness itself is informative.

## 4.4 Data Reshaping: Long to Wide and Back

Clinical data often needs to be reshaped. Lab data comes "long" (one row per test per visit) but analysis often needs it "wide" (one row per subject per visit).

### Understanding Long vs. Wide Format

**Long format** (also called "tidy" data):
```
USUBJID  VISIT  LBTEST       LBSTRESN
001-101  Week 0 Hemoglobin    13.5
001-101  Week 0 Glucose        95
001-101  Week 4 Hemoglobin    13.2
001-101  Week 4 Glucose        98
```

**Wide format**:
```
USUBJID  VISIT  Hemoglobin  Glucose
001-101  Week 0       13.5       95
001-101  Week 4       13.2       98
```

data("dm")

# Goal: Create an analysis dataset with baseline and change from baseline

# Step 1: Start with long-format SDTM LB
analysis_lb <- lb %>%
  select(USUBJID, LBDTC, VISIT, LBTEST, LBSTRESN, LBSTRESU) %>%

  # Step 2: Create baseline flag (first visit with non-missing value)
  group_by(USUBJID, LBTEST) %>%
  arrange(LBDTC) %>%
  mutate(
    LBBLFL = if_else(row_number() == 1 & !is.na(LBSTRESN), "Y", "N"),
    BASELINE_VAL = if_else(LBBLFL == "Y", LBSTRESN, NA_real_)
  ) %>%
  fill(BASELINE_VAL, .direction = "down") %>%
  ungroup() %>%

  # Step 3: Calculate change from baseline
  mutate(
    CHG = if_else(!is.na(BASELINE_VAL) & !is.na(LBSTRESN),
                  LBSTRESN - BASELINE_VAL,
                  NA_real_)
  ) %>%

  # Step 4: Convert to wide format (one row per subject per visit per LBTEST group)
  select(-LBBLFL) %>%
  pivot_wider(
    id_cols = c(USUBJID, LBDTC, VISIT),
    names_from = LBTEST,
    values_from = c(LBSTRESN, BASELINE_VAL, CHG),
    names_sep = "_"
  ) %>%

  # Step 5: Join with treatment information
  left_join(
    dm %>% select(USUBJID, ARM, AGE, SEX),
    by = "USUBJID"
  ) %>%

  # Step 6: Order columns logically
  select(USUBJID, ARM, AGE, SEX, LBDTC, VISIT, everything())

print(head(analysis_lb, n = 5))

# This creates one row per subject per visit with all labs wide
# and baseline/change calculated for each test

dm_dups <- dm %>%
  group_by(USUBJID) %>%
  filter(n() > 1)
cat("Duplicate subjects in DM:", n_distinct(dm_dups$USUBJID), "\n\n")

# STEP 4: Clean adverse event terms
cat("STEP 4: Standardizing AE Terms\n")
ae_clean <- ae %>%
  mutate(
    AETERM = str_to_upper(str_trim(AETERM)),
    AESEV = str_to_upper(str_trim(AESEV))
  ) %>%
  # Flag unusual severity values
  mutate(
    AESEV_VALID = AESEV %in% c("MILD", "MODERATE", "SEVERE", "LIFE THREATENING"),
    FLAG_INVALID_SEVERITY = if_else(!AESEV_VALID, "Y", "N")
  )

cat("Invalid severity values flagged:", sum(ae_clean$FLAG_INVALID_SEVERITY == "Y"), "\n\n")

# STEP 5: Lab data quality checks
cat("STEP 5: Lab Data Quality\n")
lb_clean <- lb %>%
  mutate(
    LBVAL_MISSING = is.na(LBSTRESN),
    # Flag clinically implausible values
    FLAG_IMPLAUSIBLE = case_when(
      LBTEST == "Glucose" & (LBSTRESN < 30 | LBSTRESN > 500) ~ "Y",
      LBTEST == "Hemoglobin" & (LBSTRESN < 5 | LBSTRESN > 20) ~ "Y",
      TRUE ~ "N"
    )
  )

cat("Lab records with implausible values:", sum(lb_clean$FLAG_IMPLAUSIBLE == "Y"), "\n\n")

# STEP 6: Build analysis dataset
cat("STEP 6: Building Analysis Dataset\n")

analysis_data <- dm %>%
  # Only include subjects with valid enrollment
  filter(!is.na(RFSTDTC)) %>%
  # Add AE summary
  left_join(
    ae_clean %>%
      filter(FLAG_INVALID_SEVERITY == "N") %>%
      group_by(USUBJID) %>%
      summarise(
        N_AE = n(),
        N_SEVERE_AE = sum(AESEV == "SEVERE", na.rm = TRUE),
        .groups = "drop"
      ),
    by = "USUBJID"

  # Add baseline labs
  left_join(
    lb_clean %>%
      filter(FLAG_IMPLAUSIBLE == "N", !LBVAL_MISSING) %>%
      filter(VISIT == "BASELINE") %>%
      select(USUBJID, LBTEST, LBSTRESN) %>%
      pivot_wider(
        id_cols = USUBJID,
        names_from = LBTEST,
        values_from = LBSTRESN,
        names_prefix = "BL_"
      ),
    by = "USUBJID"
  ) %>%
  # Replace NA counts with 0 (no adverse events recorded)
  mutate(
    N_AE = replace_na(N_AE, 0),
    N_SEVERE_AE = replace_na(N_SEVERE_AE, 0)

# STEP 8: Export flagged records for review
flags_for_review <- ae_clean %>%
